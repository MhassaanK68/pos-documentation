Ramen POS System API

A comprehensive backend API for a Ramen Shop Point-of-Sale (POS) system. This API handles menu management, order processing, user authentication, and role-based access control.

Base URL

https://house-of-ramen-api-server.onrender.com/

Authentication

JWT Bearer Token

User Roles

Role Permissions
Cashier Create orders, view personal orders and stats
Manager All cashier permissions + menu management, view all orders/users
Admin Full system access including user management

> Login Endpoint

POST  /api/auth/login

Authenticates user and returns JWT token

Parameters:
  • Body: username (string, required), password (string, required, min 6 chars)
Sample Request:
{
    "username": "cashier1",
    "password": "password123"
}
Returned Value:
{
  "success": true,
  "message": "Login successful",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
      "username": "cashier1",
      "role": "cashier",
      "is_active": true
    }
  }
}

> Update Stock

PUT  /api/stock/:id Manager+

Update the current stock level for a specific menu item

Parameters:
  • Headers: Authorization: Bearer <token>
  • Path: id (integer, required) - Menu item ID
  • Body: stock (integer, required, min: 0) - New stock quantity
Sample Request:
PUT /api/stock/1
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Content-Type: application/json

{
"stock": 25
}
Returned Value:
{
"success": true,
"message": "Stock updated successfully",
"data": {
    "menu_item": {
        "id": 1,
        "name": "Chicken Burger",
        "current_stock": 25
    }
}
}
Error Responses:
// 400 Bad Request - Stock cannot be negative
{
"success": false,
"message": "Stock cannot be negative"
}

// 404 Not Found - Menu item not found
{
"success": false,
"message": "Menu item not found"
}
